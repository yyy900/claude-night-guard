★ claude-night-guard

Safety guardrails for Claude Code running unattended.

GitHub → Threat model → Contributing →

The problem

Unattended Claude has no one to catch rm -rf ~/project, git push --force onto main, an accidental git add .env, or a curl … | bash on a typo'd domain. By the time you check in the morning, the damage is done.

This is a small set of hooks that block those tool calls before they execute, feed a structured reason back to Claude so it self-recovers, and write a JSONL audit log so you can see what hit the wall.

What it blocks

rm -rfblock-rm-rfPreToolUse · Bash

curl … | bash and friendsblock-curl-pipe-shPreToolUse · Bash
Secrets in the HEAD commit (alert only)audit-commit-secretsPostToolUse · Bash
≥3 blocks in one sessionhalt-on-repeated-blocksUserPromptSubmit

How it works

Atomic block

PreToolUse fires before the tool executes. The dangerous command never runs, so there is nothing to roll back.

Self-recovery

The hook writes a structured reason to stderr. Claude Code feeds it back as the tool's result, so the agent sees [NIGHT-GUARD BLOCKED] reason / suggestion / override and re-routes.

Auditable

Every block appends one JSONL line to ~/.claude/night-guard/incidents.jsonl. Run ./view-incidents.sh 24h in the morning.

Halt on loops

After 3 blocks in one session, UserPromptSubmit injects a halt directive so the agent stops and summarizes instead of grinding all night against the same wall.

Install

# clone, install hooks into ~/.claude/settings.json (backup saved to .bak)
git clone https://github.com/yyy900/claude-night-guard
cd claude-night-guard
./install.sh overnight-strict

# in the morning
./view-incidents.sh 24h

# remove
./uninstall.sh

No external dependencies — Bash, python3 stdlib, standard unix tools. Re-running install.sh is idempotent.

Overriding a block

When a block is a false positive and the agent needs the operation, it re-runs with an explicit ack prefix. The override is still logged.

NIGHT_GUARD_ACK=rm-rf      rm -rf ./build-cache
NIGHT_GUARD_ACK=force-push git push --force-with-lease origin feature
NIGHT_GUARD_ACK=curl-pipe  curl https://get.pnpm.io/install.sh | bash

For block-out-of-cwd-edit: NIGHT_GUARD_ALLOW_PATHS=/path/one:/path/two extends the allowed-prefix list.

What it does not do

  • It does not catch the model misjudging what to build — only what tools to invoke. Swallowed exceptions, silent fallbacks, scope creep: those need CLAUDE.md rules and post-hoc review.
  • It does not auto-rollback. PostToolUse alerts you to a bad commit; rolling back is itself destructive and stays a human decision.
  • It does not enforce cost caps. Use provider rate limits and DISABLE_NON_ESSENTIAL_MODEL_CALLS.
  • It is not a security boundary against a compromised model. If the model's output is attacker-controlled, the same output can write bypasses. Real isolation lives in the OS sandbox.
